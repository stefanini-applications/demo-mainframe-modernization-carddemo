CICS Batch COBOL VSAM TRANFILE CT00 · CT01 · CT02

📋 Module Overview

Purpose: List, view, and manually add transactions online; post transactions in batch (POSTTRAN), generate reports (TRANREPT), and produce statements (CREASTMT).

Business Context: Transactions represent all financial activity on a card. Transaction records feed account balance updates, interest calculations (via TCATBALF), and reporting. Batch posting is the critical nightly job that makes transactions permanent.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • View transaction list: As a cardholder, I want to view my recent transactions so I can monitor for unauthorized charges
  • View transaction detail: As a cardholder, I want to see merchant name, city, and amount so I can verify individual charges
  • Add transaction: As a cardholder, I want to add a manual transaction entry so I have a record of a purchase made offline
  • Filter transactions: As a cardholder, I want to filter transactions by date range so I can review a specific period
  • Batch posting: As a system, I want to run nightly POSTTRAN so account balances are updated with all processed transactions
  • Transaction report: As a system operator, I want to generate the daily transaction report so I can reconcile financial totals

⚡ Development Acceleration Factors

  • TRANFILE AIX on TRAN-CARD-NUM: Alternate index already exists — transaction list by card is efficient and ready to use
  • CVTRA05Y copybook: Full TRAN-RECORD structure defined — new fields follow established pattern
  • POSTTRAN batch template: CBTRN02C handles all balance updates — extend for new transaction types without redesign
  • COMBTRAN SORT job: Combines transaction files before posting — supports multi-source transaction feeds

📊 Complexity Guidelines

Simple (1–2 pts) Add merchant field to detail view; change sort order of list; display transaction amount with color coding
Medium (3–5 pts) Add date range filter to transaction list; add transaction category description lookup; add total amount display to list screen
Complex (5–8 pts) New transaction type with different posting logic in CBTRN02C; transaction reversal/void capability; real-time balance update (bypassing batch); new TRANFILE field requiring RECLN change

🔧 Technical Foundation

Transaction List Flow

Entry: CARD-NUM in COMMAREA (from Card or Account module)
  → Read TRANFILE (AIX key = TRAN-CARD-NUM) sequentially
  → Paginate results (PF7=scroll up, PF8=scroll down)
  → Display: TRAN-ID, TRAN-TYPE-CD, TRAN-AMT, TRAN-ORIG-TS
  → User selects transaction → TRAN-ID to COMMAREA
  → Navigate to Transaction Detail (CT01)

Transaction Add Flow

Entry: CT02 transaction, CARD-NUM in COMMAREA
  → Display blank COTRN02 screen
  → User enters: type, category, amount, merchant info, description
  → Validate: amount format, type/category codes, card exists
  → Generate TRAN-ID (system-assigned unique key)
  → Set TRAN-ORIG-TS = current timestamp
  → WRITE to TRANFILE
  → Display confirmation

POSTTRAN Batch Flow (CBTRN02C)

Read TRANFILE sequentially (unposted transactions)
  For each transaction:
    → Read XREFFILE (key=TRAN-CARD-NUM) → get ACCT-ID
    → Read ACCTFILE (key=ACCT-ID) for current balance
    → Update ACCT-CURR-BAL += TRAN-AMT
    → Update ACCT-CURR-CYC-DEBIT or ACCT-CURR-CYC-CREDIT
    → Construct TRAN-CAT-KEY (ACCT-ID + TYPE + CAT)
    → Read/update TCATBALF for category balance
    → Set TRAN-PROC-TS = processing timestamp
    → REWRITE TRANFILE (mark as posted)

Data Model

TRAN-RECORD (TRANFILE, RECLN=350):
  TRAN-ID              PIC X(16)        ← Primary key
  TRAN-TYPE-CD         PIC X(02)        ← e.g. 'SA' (sale), 'CR' (credit)
  TRAN-CAT-CD          PIC 9(04)        ← Category for interest grouping
  TRAN-SOURCE          PIC X(10)        ← Source system
  TRAN-DESC            PIC X(100)       ← Description
  TRAN-AMT             PIC S9(09)V99    ← Negative=debit, Positive=credit
  TRAN-MERCHANT-ID     PIC 9(09)
  TRAN-MERCHANT-NAME   PIC X(50)
  TRAN-MERCHANT-CITY   PIC X(50)
  TRAN-MERCHANT-ZIP    PIC X(10)
  TRAN-CARD-NUM        PIC X(16)        ← AIX key
  TRAN-ORIG-TS         PIC X(26)        ← Original timestamp
  TRAN-PROC-TS         PIC X(26)        ← Set on POSTTRAN run

🔗 Dependencies

  • credit-cards: CARD-NUM in COMMAREA required for transaction list filter
  • accounts: POSTTRAN updates ACCTFILE — account module data affected by every posting run
  • TRANFILE AIX: TRAN-CARD-NUM alternate index must be defined and built (TRANIDX JCL job)
  • TCATBALF: POSTTRAN updates category balances — feeds interest calculation (INTCALC)
  • CLOSEFIL/OPENFIL: POSTTRAN requires CICS files closed before run

✅ Acceptance Criteria Examples

Transaction list — shows recent transactions:
Given a card with transactions in TRANFILE,
When navigating to Transaction List (CT00),
Then transactions are displayed in reverse chronological order with ID, type code, amount, and date
Transaction detail:
Given a selected transaction ID,
When navigating to Transaction Detail (CT01),
Then merchant name, city, ZIP, description, type code, category, and amount are all displayed
Add transaction — success:
Given a valid card in COMMAREA and all required fields entered,
When submitting Transaction Add (CT02),
Then a new TRAN-RECORD is written to TRANFILE with a system-generated TRAN-ID and current timestamp
Add transaction — validation failure:
Given an invalid amount format or missing required field,
When pressing ENTER on CT02,
Then the screen redisplays with field-level error and no TRANFILE write occurs
POSTTRAN batch — success:
Given unposted transactions in TRANFILE,
When CBTRN02C completes with RC=0,
Then each account's ACCT-CURR-BAL is updated; TRAN-PROC-TS is set on all processed records; TCATBALF balances reflect totals
Pagination:
Given more than one page of transactions,
When pressing PF8 (scroll down) or PF7 (scroll up),
Then the next/previous page of transactions is displayed without data loss
