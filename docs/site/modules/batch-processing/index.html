Overview & Architecture

The batch-processing module encompasses all z/OS batch jobs and COBOL programs that run outside the CICS online region. It is responsible for the nightly and monthly processing cycles that keep CardDemo data consistent: posting daily transactions, calculating interest, generating statements, and refreshing VSAM master files from source data.

Batch programs are invoked via JCL job streams on z/OS. They access the same VSAM KSDS files used by CICS online programs. Because VSAM files cannot be simultaneously open for update by both CICS and batch, a CLOSEFIL/OPENFIL protocol is required: CICS files are closed before batch runs and reopened afterwards.

Architecture Pattern: Mainframe batch — COBOL programs invoked by JCL on z/OS, reading and updating VSAM KSDS files. GDG (Generation Data Groups) datasets are used for versioned outputs (rejected transactions, reports, system-generated transaction files).

System Position

Batch processing sits downstream of the CICS online module: transactions entered online are accumulated in a daily transaction file (DALYTRAN) and then posted by the POSTTRAN batch job. It also runs independently for periodic tasks such as monthly interest calculation (INTCALC) and statement printing (CREASTMT).

Business Rules

File Access Protocol (CICS/Batch Coordination)

Critical: CICS files must be closed (CLOSEFIL job) before running any batch job that updates VSAM files shared with the CICS online region. After batch completion, OPENFIL must be run to restore CICS access. Skipping this protocol risks VSAM file status 97 (already open) errors and potential data corruption.
  • CLOSEFIL issues: CEMT SET FIL(TRANSACT) CLO, CEMT SET FIL(CCXREF) CLO, CEMT SET FIL(ACCTDAT) CLO, CEMT SET FIL(CXACAIX) CLO, CEMT SET FIL(USRSEC) CLO to CICS region CICSAWSA
  • OPENFIL opens the same files using CEMT SET FIL(...) OPE commands via SDSF
  • Read-only extract programs (CBACT02C, CBACT03C, CBCUS01C) may not require file closure

Transaction Posting Rules (CBTRN02C)

  • Reads DALYTRAN.PS sequentially; validates card number via XREFFILE lookup (random read on CARD-NUM key)
  • Looks up account from XREFFILE (XREF-ACCT-ID) and reads ACCTFILE for current balance
  • Updates ACCT-CURR-BAL by adding TRAN-AMT (negative amounts reduce balance for debits)
  • Updates ACCT-CURR-CYC-CREDIT or ACCT-CURR-CYC-DEBIT based on sign
  • Inserts validated record into TRANFILE (indexed, random write on TRAN-ID key)
  • Upserts TCATBALF record keyed by ACCT-ID || TYPE-CD || CAT-CD
  • Invalid card numbers or account lookup failures: record written to DALYREJS GDG(+1) with rejection reason
  • Job completes with return code 0000 on success; non-zero indicates I/O or data error

Interest Calculation Rules (CBACT04C)

  • Accepts PARM=YYYYMMDD (billing cycle processing date)
  • Reads TCATBALF sequentially (all records for all accounts)
  • For each TCATBALF record, constructs DISCGRP key: ACCT-GROUP-ID || TRAN-TYPE-CD || TRAN-CAT-CD
  • Interest = TRAN-CAT-BAL multiplied by rate from DISCGRP record
  • Writes one system-generated interest transaction per TCATBALF record to TRANSACT sequential file (SYSTRAN GDG+1)
  • Updates ACCTFILE current balance with computed interest amount
  • After INTCALC completes, COMBTRAN must merge SYSTRAN(0) + TRANSACT.BKUP(0) and reload TRANSACT VSAM

Transaction Report Rules (CBTRN03C)

  • TRANREPT job runs in three phases: backup TRANFILE → SORT filter by date range + sort by card → CBTRN03C print
  • Date range specified in JCL SYMNAMES: PARM-START-DATE and PARM-END-DATE in format YYYY-MM-DD
  • CBTRN03C reads DATEPARM sequential file for additional filtering parameters
  • Type descriptions looked up from TRANTYPE VSAM (keyed by TRAN-TYPE-CD)
  • Category descriptions looked up from TRANCATG VSAM (keyed by TYPE-CD || CAT-CD)
  • Output TRANREPT dataset: LRECL=133, RECFM=FB, GDG(+1)
  • Report can be triggered from CICS via CORPT00C program (CR00 transaction)

Statement Generation Rules (CBSTM03A/B)

  • CBSTM03A drives all processing; calls CBSTM03B as a subroutine via COBOL CALL statement
  • Produces two concurrent output formats: plain-text (STMTFILE, LRECL=80) and HTML (HTMLFILE, LRECL=100)
  • Input TRNXFILE read sequentially and grouped by card number (WS-SAVE-CARD work field)
  • Running totals per card maintained in WS-TOTAL-AMT (COMP-3 packed decimal)
  • Uses mainframe control block addressing, ALTER/GO TO, COMP and COMP-3 variables, and 2-dimensional arrays
  • Uses COSTM01, CVACT03Y, CUSTREC, CVACT01Y copybooks for data structures

GDG Dataset Management Rules

  • GDG bases defined by DEFGDGB and DEFGDGD jobs using IDCAMS DEFINE GDG
  • Generation notation: GDG(+1) = new generation (output), GDG(0) = current, GDG(-1) = previous
  • GDG datasets: DALYREJS, TRANSACT.BKUP, SYSTRAN, TRANSACT.DALY, TRANSACT.COMBINED, TRANREPT
  • Prior generations retained for audit trail and potential reruns

Import and Export Rules

  • CBIMPORT reads multi-record indexed file (EXPFILE); record type field identifies CUSTOMER, ACCOUNT, XREF, or TRANSACTION data
  • Checksum validation performed per imported record group; failures written to error report with statistics
  • CBEXPORT reads all five normalized files and produces a single multi-record typed export file
  • Both programs output processing statistics: total records read, written, and rejected

Processing Flows

Nightly Transaction Posting

  1. Submit CLOSEFIL — CEMT SET FILE CLOSE for TRANSACT, ACCTDAT, CCXREF, CXACAIX via SDSF to CICS region CICSAWSA
  2. Submit POSTTRAN (CBTRN02C) — reads DALYTRAN.PS; posts to TRANFILE VSAM; updates ACCTFILE and TCATBALF; rejects to DALYREJS GDG(+1)
  3. Submit OPENFIL — CEMT SET FILE OPEN restores CICS access
  4. (Optional) Submit WAITSTEP (COBSWAIT) — insert delay before next dependent job

Monthly Interest Calculation

  1. Submit CLOSEFIL to close CICS VSAM files
  2. Submit TRANBKP — backup TRANSACT VSAM to TRANSACT.BKUP GDG(+1) via IDCAMS REPRO
  3. Submit INTCALC (CBACT04C, PARM=YYYYMMDD) — read TCATBALF, compute interest per DISCGRP, update ACCTFILE, write interest transactions to SYSTRAN GDG(+1)
  4. Submit COMBTRAN — SORT merge TRANSACT.BKUP(0) + SYSTRAN(0) by TRAN-ID; IDCAMS REPRO combined file to TRANSACT VSAM
  5. Submit OPENFIL to restore CICS access
  6. Submit CREASTMT (CBSTM03A/B) — generate plain-text and HTML statements for all accounts

Transaction Report Generation

  1. Update date range in JCL SYMNAMES (PARM-START-DATE, PARM-END-DATE)
  2. IDCAMS REPRO unloads TRANSACT.VSAM.KSDS to TRANSACT.BKUP GDG(+1)
  3. SORT step: filter by date range, sort by TRAN-CARD-NUM ascending; output to TRANSACT.DALY GDG(+1)
  4. CBTRN03C reads TRANSACT.DALY(+1), looks up type/category descriptions, writes TRANREPT GDG(+1)

Operations Guide

Nightly Batch Checklist

  1. Verify DALYTRAN.PS dataset has been created by online CICS session or upstream data feed
  2. Submit CLOSEFIL — confirm all files in CLO status via CICS CEMT INQUIRE FILE
  3. Submit POSTTRAN — verify CC=0000; check DALYREJS GDG(+1) record count (should be 0 for clean run)
  4. Submit OPENFIL — confirm all files in OPE status in CICS
  5. Review SYSPRINT output for record counts, processing totals, and any error messages

Monthly Batch Checklist

  1. Confirm nightly POSTTRAN completed for all business days in the billing cycle
  2. Submit CLOSEFIL
  3. Submit TRANBKP — backup TRANSACT VSAM to GDG; verify record count
  4. Submit INTCALC with PARM=YYYYMMDD (last day of billing cycle)
  5. Submit COMBTRAN — merge interest transactions; verify combined record count
  6. Submit OPENFIL
  7. Submit CREASTMT — generate statements; verify both STMTFILE and HTMLFILE output
  8. Submit TRANREPT — update date range in JCL SYMNAMES first; verify TRANREPT GDG(+1) output
Expected Outcomes per Job:
  • POSTTRAN: All DALYTRAN records posted; DALYREJS GDG(+1) empty or only invalid-card rejects; ACCTFILE balances updated
  • INTCALC: SYSTRAN GDG(+1) has one interest transaction per TCATBALF record; ACCTFILE balances updated with interest
  • CREASTMT: STMTFILE and HTMLFILE written for all accounts with transactions in the cycle
  • TRANREPT: TRANREPT GDG(+1) contains page totals, account totals, and grand total for the date range

Failure Handling

ScenarioSymptomResolution
VSAM file not closed before batch update File status 97 (already open); job abend S013 or S0C4 Run CLOSEFIL; verify CICS file status via CEMT INQUIRE FILE; check no active transactions
XREFFILE card not found in DALYTRAN Reject record in DALYREJS GDG(+1); count shown in SYSPRINT Investigate orphaned card numbers; run XREFFILE refresh to reload from source
DISCGRP key missing for account group CBACT04C writes error to SYSOUT; interest not computed for that TCATBALF record Add missing DISCGRP record for the group/type/category combination; rerun INTCALC
GDG(+1) allocation failure JCL allocation error before program executes; job fails in allocation phase Verify GDG base defined (run DEFGDGB/D); increase LIMIT on GDG base if at maximum generations
DALYTRAN.PS not found POSTTRAN fails with JCL NOT CATALOGED or IEC150I 913 Verify upstream feed or CICS export job created the dataset for the current business day

User Stories and Acceptance Criteria

Nightly Transaction Posting

As a system operator, I want to run nightly transaction posting so account balances are updated each business day.

Acceptance Criteria:

  • POSTTRAN job completes with return code 0000
  • All DALYTRAN records either posted to TRANFILE or written to DALYREJS with a rejection reason
  • ACCTFILE ACCT-CURR-BAL reflects cumulative sum of all posted transaction amounts
  • TCATBALF records updated for each account/type/category combination processed
  • CICS files closed before job and reopened after (no active user sessions interrupted)

Monthly Interest Calculation

As a system operator, I want to run monthly interest calculation so account statements reflect accrued interest charges.

Acceptance Criteria:

  • INTCALC job accepts PARM=YYYYMMDD for the billing cycle date
  • One interest transaction written to SYSTRAN GDG(+1) per TCATBALF record processed
  • Interest amount = category balance multiplied by rate from matching DISCGRP record
  • ACCTFILE balances updated with computed interest amounts
  • COMBTRAN successfully merges SYSTRAN(0) and TRANSACT.BKUP(0) into TRANSACT VSAM

Account Statement Generation

As a system operator, I want to run monthly statement generation so all customers receive accurate billing summaries.

Acceptance Criteria:

  • CREASTMT produces both plain-text (STMTFILE, LRECL=80) and HTML (HTMLFILE, LRECL=100) output
  • Each account with transactions has a corresponding statement section with individual line items
  • Statement totals reconcile to sum of individual transaction amounts per card
  • Job completes within the 4-hour monthly batch window

VSAM Master File Refresh

As a system operator, I want to refresh VSAM master files from source data so the test environment is reset to a known state.

Acceptance Criteria:

  • ACCTFILE, CARDFILE, CUSTFILE, TRANFILE, XREFFILE, TCATBALF, DISCGRP refresh jobs complete CC=0000
  • Record counts in refreshed VSAM files match source sequential dataset counts
  • Alternate indexes (XREFFILE AIX) rebuilt successfully after REPRO
  • CICS files closed before refresh and reopened after

Branch Migration Data Import

As a system operator, I want to import branch migration data so new customers and accounts are loaded into CardDemo.

Acceptance Criteria:

  • CBIMPORT reads EXPFILE and splits records by type to CUSTOUT, ACCTOUT, XREFOUT, TRNXOUT
  • Checksum validation passes for all successfully imported record groups
  • Error report produced for any records failing validation with reason codes
  • Import statistics (records read, written, rejected) written to SYSPRINT

Data Integrity: CICS File Coordination

As a system, I want CICS files closed before batch updates so data integrity is maintained between online and batch processing.

Acceptance Criteria:

  • CLOSEFIL successfully closes all five CICS files: TRANSACT, CCXREF, ACCTDAT, CXACAIX, USRSEC
  • CEMT INQUIRE FILE shows CLO status for all files before batch programs execute
  • OPENFIL successfully reopens all files after batch completion
  • No CICS transaction failures due to file unavailability during the maintenance window
