User Story Development Guide — CardDemo Mainframe Application

CICS Batch COBOL VSAM ACCTFILE CAVW · CAUP

📋 Module Overview

Purpose: View and update credit card account details online; run interest calculations and data extract/reporting in batch.

Business Context: The account is the central financial entity. All cards, transactions, billing, and interest are linked to an account. The account record holds balance, credit limit, and group ID for interest rate determination.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • View account: As a cardholder, I want to view my account balance, credit limit, and status so I know my current financial position
  • Update address: As a cardholder, I want to update my billing ZIP code so my address on file is accurate
  • View cycle data: As a cardholder, I want to see current cycle credits and debits so I can track my billing cycle activity
  • Interest calc: As a system, I want to calculate monthly interest per account-group-type-category so balances reflect accrued interest
  • Account status: As an administrator, I want to change an account's active status so access can be suspended when needed

⚡ Development Acceleration Factors

  • CVACT01Y copybook: Account record structure already defined — no redesign needed for standard account fields
  • XREFFILE lookup: Card-to-Account-to-Customer cross-reference already exists — multi-entity lookups are a single VSAM read chain
  • CBACT04C template: Interest calculation follows established TCATBALF × DISCGRP pattern — extend for new rate types
  • View/Update separation: CAVW (read-only) and CAUP (update) already separate — can evolve independently

📊 Complexity Guidelines

Simple (1–2 pts) Add new display field to account view screen; change field order in BMS map; add read-only field to update screen
Medium (3–5 pts) Add new updatable account field with validation; modify interest formula for a new rate tier; add customer info display to account view (requires XREF + CUST read)
Complex (5–8 pts) Add new account field requiring VSAM RECLN change (triggers file redefinition + all batch programs); new interest tier requiring DISCGRP schema change; account history/audit trail (new VSAM file)

🔧 Technical Foundation

Account Lookup Flow

Entry point: Card number in COMMAREA (from previous screen)
  → Read XREFFILE (key=CARD-NUM) → get ACCT-ID + CUST-ID
  → Read ACCTFILE (key=ACCT-ID)  → ACCOUNT-RECORD
  → Read CUSTFILE (key=CUST-ID)  → CUSTOMER-RECORD
  → Display on COACTVW screen

Interest Calculation Flow (CBACT04C)

Read TCATBALF sequentially (all account/type/category combos)
  For each record:
    → Read XREFFILE (AIX by ACCT-ID) → get CARD-NUM
    → Read ACCTFILE (key=ACCT-ID)    → get ACCT-GROUP-ID
    → Read DISCGRP (key=GROUP+TYPE+CAT) → get DIS-INT-RATE
    → Compute interest = TRAN-CAT-BAL × DIS-INT-RATE
    → Update ACCTFILE: ACCT-CURR-BAL += interest

Data Model (key fields)

ACCOUNT-RECORD (ACCTFILE, RECLN=300):
  ACCT-ID                 PIC 9(11)      ← Primary key
  ACCT-ACTIVE-STATUS      PIC X(01)      ← Y/N
  ACCT-CURR-BAL           PIC S9(10)V99  ← Current balance
  ACCT-CREDIT-LIMIT       PIC S9(10)V99
  ACCT-CASH-CREDIT-LIMIT  PIC S9(10)V99
  ACCT-OPEN-DATE          PIC X(10)      ← YYYY-MM-DD
  ACCT-EXPIRAION-DATE     PIC X(10)
  ACCT-CURR-CYC-CREDIT    PIC S9(10)V99  ← Cycle credits (payments)
  ACCT-CURR-CYC-DEBIT     PIC S9(10)V99  ← Cycle debits (charges)
  ACCT-ADDR-ZIP           PIC X(10)
  ACCT-GROUP-ID           PIC X(10)      ← Links to DISCGRP interest rates

🔗 Dependencies

  • authentication: COMMAREA must contain valid CDEMO-USER-ID before account access
  • ACCTFILE VSAM: Primary account data store — must be loaded before online use
  • XREFFILE VSAM: Required for card-to-account lookup chain
  • CUSTFILE VSAM: Customer name displayed on account view screen
  • TCATBALF + DISCGRP: Required by CBACT04C interest calculation job
  • CLOSEFIL/OPENFIL JCL: Batch account jobs require CICS file closure first

✅ Acceptance Criteria Examples

Account view — success:
Given a signed-in cardholder with a valid account,
When navigating to Account View (CAVW),
Then current balance, credit limit, expiry date, and active status are displayed from ACCTFILE; customer name displayed from CUSTFILE
Account update — ZIP code:
Given a signed-in cardholder on Account Update (CAUP),
When entering a valid 5-digit ZIP and pressing ENTER,
Then ACCT-ADDR-ZIP in ACCTFILE is updated and a success message is displayed
Account update — validation failure:
Given an invalid input (non-numeric ZIP, blank required field),
When pressing ENTER,
Then the screen redisplays with a field-level error message and no VSAM update occurs
Interest calculation (INTCALC batch):
Given TCATBALF and DISCGRP are populated,
When CBACT04C runs to completion (RC=0),
Then each account's ACCT-CURR-BAL is increased by computed interest; TCATBALF record count matches input
Inactive account:
Given an account with ACCT-ACTIVE-STATUS = 'N',
When viewing the account,
Then the status field displays 'Inactive' and transaction add/payment options are disabled
